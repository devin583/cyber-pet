📐 我想自己做一个形象,需要准备几张图、什么样的图?

硬性规格(每张图都要满足):

  • 格式:PNG / SVG / GIF / WebP,透明背景
  • 尺寸:正方形,建议 512×512 以内(显示时约 128px);所有帧必须同尺寸、同机位、同大小
  • 构图:角色正面全身、居中,脚底贴住画布底边(程序把底边踩在任务栏上)

准备几张?三档任选:

档位张数要画的内容
能跑1 张待机(睁眼闭嘴)。其余动作自动用它顶替
推荐4 张待机 + 闭眼(供眨眼/睡觉)+ 张嘴(供说话/吃饭)+ 笑眯眼(供开心)
完整约 9 张再加:呼吸第 2 帧、走路左右 2 帧、被拎起(惊讶)、落地压扁、被戳(不爽)

只有 1~4 张时,项目自带脚本 scripts/make_pack_from_image.py 可自动烘焙出走路摇摆、拎起拉伸、落地压扁等全套动作,无需逐帧作画。

① 定时提醒(如每天 18:00 提醒下班)

        
Windows:把下面一行存为 remind.ps1,再用任务计划设定时执行:

        
Invoke-RestMethod "http://127.0.0.1:12580/notify?title=下班啦&body=收拾收拾回家吃饭!"

        
注册任务(管理员 PowerShell 执行一次):

        
$action  = New-ScheduledTaskAction -Execute "powershell" -Argument "-WindowStyle Hidden -File C:\scripts\remind.ps1"
$trigger = New-ScheduledTaskTrigger -Daily -At 18:00
Register-ScheduledTask -TaskName "PetRemind" -Action $action -Trigger $trigger

        
macOS / Linux 用 crontab:

        
0 18 * * * curl -s "http://127.0.0.1:12580/say?text=下班啦"


        
② 订阅内容(RSS / UP 主更新)

        
思路:本机跑一个轮询脚本,发现新内容就调 /notify。
           下面是零依赖的 Python 示例(每 10 分钟查一次,存为 rss_watch.py 后台运行):

        
import time, urllib.request, urllib.parse, xml.etree.ElementTree as ET

FEED = "https://sspai.com/feed"          # 换成你的订阅源
PET  = "http://127.0.0.1:12580"
seen = set()

while True:
    try:
        xml = urllib.request.urlopen(FEED, timeout=10).read()
        for item in ET.fromstring(xml).iter("item"):
            title = item.findtext("title", "")
            if title and title not in seen:
                if seen:                  # 首轮只记录不打扰
                    q = urllib.parse.urlencode({"title": "📰 订阅更新", "body": title})
                    urllib.request.urlopen(f"{PET}/notify?{q}", timeout=5)
                seen.add(title)
    except Exception:
        pass                              # 网络波动跳过本轮
    time.sleep(600)
